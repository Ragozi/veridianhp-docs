# Data handling

This page is a plain-English summary of what data Veridian Health Partners
collects across our products, where it lives, how long it's kept, and how
it gets deleted. It covers Veridian Health today; Veridian Billing will
follow the same principles when it launches.

## What we collect

| Data type | Source | Why we need it |
| --- | --- | --- |
| Facility account info | Dashboard | Authentication, support, billing |
| Staff user accounts | Dashboard | Role-based access, audit trail |
| Inmate/patient identifiers | Facility referral | Link consult to the correct individual |
| Clinical intake | Facility referral + session | PT assessment and documentation |
| Visit notes | Provider | Clinical record, facility handoff |
| Session metadata | Platform | Scheduling, audit, quality assurance |
| Audit log entries | Platform | Tamper-evident record of actions |
| Provider credentials | Provider onboarding | Licensure verification, compliance |

## What we do not collect

- Full medical records or external EHR exports (unless explicitly integrated
  in a future release)
- Criminal history, custody status, or disciplinary records (unless clinically
  relevant and provided by facility staff)
- Inmate personal device data — consults run on facility-controlled hardware
- Payment card or bank account data (handled by Veridian Billing when available,
  through compliant payment partners)

## Where data lives

All Veridian production data is hosted in the **United States**. Sub-processors
are US-based or operate under BAA-covered US data processing agreements.

## Retention

| Data type | Default retention |
| --- | --- |
| Consult records and visit notes | Duration of facility relationship + 6 years |
| Audit logs | 7 years |
| Provider credentialing records | Duration of provider relationship + 6 years |
| Pilot/sandbox test data | 90 days after pilot completion |

Facilities may request custom retention schedules within HIPAA-permitted
bounds. Shorter retention is supported; longer retention requires documented
justification.

## Deletion

When a facility ends its Veridian relationship:

1. Export is available for 30 days — consult records, audit logs, and account
   data in standard formats.
2. After the export window, production PHI is **cryptographically erased**
   from active systems within 90 days.
3. Backups containing the facility's PHI expire on their normal rotation
   cycle (maximum 90 days).
4. A deletion certificate is provided on request.

## Data access requests

Facilities can export their data at any time from the dashboard. Inmate
access requests (HIPAA right of access) are routed through the facility;
Veridian provides the underlying consult data to the facility's authorized
staff to fulfill the request.
