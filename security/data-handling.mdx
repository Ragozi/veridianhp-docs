# Data handling

Plain-language summary of data in **Veridian Motion**. For entity detail and
RLS, see [Data model & privacy](/veridian-motion/data-model-privacy).

## What we collect

| Data | Source | Purpose |
| --- | --- | --- |
| Patient / inmate record | Staff enrollment | Authentication, roster |
| Inmate number + PIN hash | Staff enrollment | Kiosk sign-in |
| Body-map selections | Inmate kiosk | Triage context |
| Complaint text (and voice-to-text) | Inmate kiosk | Triage context |
| Posture photos (4 views) | Inmate kiosk | PT review |
| Q&A responses | Inmate kiosk | Triage context |
| AI triage output | Server (multi-LLM) | PT review only until released |
| Exercise plan + media refs | PT review | Care delivery after review |
| PT notes | PT | Clinical record; optional inmate share |
| Messages | Staff + inmate | Monitored internal record |
| Staff accounts + roles | Admin | Dashboard access |
| Audit events | Platform | Compliance (no PHI in payload) |

## What we do not collect in logs

- Clinical narrative in application logs
- Raw photos in audit log entries
- LLM prompts/responses in standard application logs

## Where data lives

Production data is hosted in **United States** regions through Lovable Cloud /
Supabase unless an agency deploys to a dedicated environment under contract.

## Retention

Retention follows **agency agreement** and applicable health-record rules.
Typical defaults align with multi-year clinical record retention. Agencies may
request export before offboarding.

## Deletion

At contract end, agencies receive an **export window**, then Veridian Health
Partners executes agreed **deletion procedures** for PHI in managed
environments. Deletion certificates available on request.

## Email and PDF

Assessment PDFs sent to **allowlisted addresses** require a **BAA-covered
email provider** before production use with real PHI.
