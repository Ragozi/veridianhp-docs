# Security overview

Veridian Health Partners operates as a **Business Associate** under HIPAA
and is designed from the ground up for healthcare data in correctional
settings. This page is the public summary of our security posture across
all products — starting with Veridian Health.

For evidence-level detail (control matrices, audit reports, penetration
test summaries), customers and prospects can request our security package
through their account contact.

## What we protect

- **Protected Health Information (PHI):** Inmate/patient identifiers,
  assessment images, questionnaire responses, PT-driven recommendations,
  referral information, and any other data covered under HIPAA.
- **Facility credentials:** Admin accounts, staff access tokens, API keys
  (when applicable), and webhook signing secrets.
- **Provider credentials:** Licensed clinician accounts, licensure records,
  and session access.

## How we protect it

### Encryption

- **In transit:** TLS 1.2+ on every connection, including internal services
  and telehealth sessions.
- **At rest:** AES-256-GCM field-level encryption on PHI fields. Database
  storage volumes are additionally encrypted at the platform level.
- **Key management:** Encryption keys are versioned to support rotation
  without service interruption.

### Access controls

- **Least privilege.** Engineers do not have routine access to production
  PHI. Break-glass access requires approval, expires automatically, and is
  fully audited.
- **Multi-factor authentication** is required for every Veridian team member
  with access to any production system.
- **Role-based access** in product dashboards — facility administrators,
  medical staff, custody staff, and providers see only what their role
  requires.

### Audit logging

Every significant action — referral creation, session start/end,
documentation access, admin configuration changes — is recorded in a
tamper-evident audit log. Facilities can export logs for compliance review
and accreditation documentation.

### Assessment image security

Veridian Health assessment photos are captured on facility-controlled
devices, transmitted over encrypted connections, and stored with the same
field-level encryption as all other PHI. Images are never processed on
inmate personal devices or consumer-grade apps.

## Correctional-specific considerations

Healthcare in custody settings adds constraints beyond standard HIPAA:

- **Custody staff access** is limited to logistics (scheduling, escort) —
  not clinical notes unless explicitly authorized.
- **Device control** — consults run on facility hardware, not inmate personal
  devices, reducing data exfiltration risk.
- **Minimum necessary** — we collect only the PHI required for the consult
  and documentation, not full medical records by default.

## Compliance posture

Every facility using Veridian Health for production signs a **Business
Associate Agreement (BAA)** before live PHI flows. Provider partners operate
under platform agreements that flow down HIPAA obligations.

Sub-processors that touch PHI each hold a signed BAA with Veridian Health
Partners. The current sub-processor list is available to customers on request.
