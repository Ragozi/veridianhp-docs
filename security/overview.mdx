# Security overview

Veridian Health Partners designs **Veridian Motion** with healthcare and
correctional use cases in mind. This page summarizes the **security posture**
honestly — what the application implements and what **hosting agencies** must
provide.

<Warning>
  Veridian Motion is **not HIPAA-certified**, **CJIS-authorized**, or
  **FedRAMP-authorized**. HIPAA-aligned design ≠ certification. Agencies
  choose hosting and operational controls to meet their obligations.
</Warning>

## What Veridian Motion enforces

### Access control

- **Staff:** Email, password, mandatory TOTP MFA; role-based UI and API
- **Inmates:** Inmate number + PIN; access limited to own data via RLS
- **Roles:** `pt`, `admin`, `officer`, `warden`, `it_admin` — granted only by admin

### Data protection

- **Row-Level Security** on every Postgres table
- **Private photo storage** with five-minute signed URLs after authorization
- **No PII in browser localStorage** for inmates; in-memory session on kiosk
- **No PHI in application logs**; append-only audit log for sensitive actions

### Application hardening (kiosk)

- Blocked external navigation, strict CSP, self-hosted fonts, idle auto-reset
- Internal messaging only — no external egress for inmate communications

### Clinical safety gate

- AI output is **non-diagnostic** and **PT-reviewed** before inmate release

## What agencies / facilities must provide

| Requirement | Owner |
| --- | --- |
| BAA with health provider and PHI-touching vendors (email, etc.) | Agency + Veridian Health Partners |
| CJIS-compliant or FedRAMP / GovCloud hosting (if required) | Agency |
| MDM kiosk lockdown, device supervision | Agency IT + custody |
| Network segmentation and firewall allowlists | Agency IT |
| Personnel screening and security training | Agency |
| Physical security of kiosk location | Agency |

## Encryption

- **In transit:** TLS for all client and service connections
- **At rest:** Supabase / cloud provider encryption for database and storage volumes
- **Photos:** Private bucket; no public ACLs

## Audit and monitoring

Sensitive actions write to an **append-only audit log** (enroll, assign, review,
notes, roles, MFA reset, messages, email). Application logs exclude PHI.

Agencies may export audit data for compliance review from the dashboard
(`/audit`, `/compliance`, `/security` — role-scoped).
