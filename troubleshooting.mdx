# Troubleshooting

Most issues fall into a few categories: access and accounts, consult device
setup, live session problems, and documentation. Start with the symptom that
matches your situation.

## Access and accounts

**I can't log in to the dashboard.**
- Confirm you're using the email address provisioned during onboarding.
- Reset your password from the login page.
- If your account was recently created, allow up to 15 minutes for
  provisioning to complete.
- Contact [support@veridianhp.com](mailto:support@veridianhp.com) if the
  issue persists.

**A staff member can't see consult notes.**
- Check their role assignment — clinical notes require medical staff or
  administrator access, not custody-only roles.
- Administrators can adjust roles in **Settings → Users**.

**Our facility isn't showing in the provider queue.**
- Confirm onboarding is complete: BAA signed, org provisioned, at least one
  consult endpoint configured.
- Contact your Veridian account contact if setup shows complete but the
  facility isn't visible.

## Consult device setup

**The consult device can't connect to Veridian Health.**
- Verify outbound HTTPS (port 443) is allowed on the facility network.
- Confirm the device meets the setup checklist provided during onboarding.
- Try the device on a different network segment to isolate firewall rules.
- Run a test session with Veridian support before scheduling inmate consults.

**Audio or video quality is poor during sessions.**
- Check network bandwidth at the consult location — telehealth requires
  stable connectivity.
- Close other applications on the consult device.
- Ensure the device camera and microphone permissions are granted to the
  browser or app.
- If quality remains poor, document the issue and reschedule rather than
  conducting an incomplete assessment.

## Live session issues

**The provider didn't join at the scheduled time.**
- Check the session status in the dashboard — the provider may have been
  reassigned or the session rescheduled.
- Contact [support@veridianhp.com](mailto:support@veridianhp.com) with the
  session ID for immediate escalation.

**The session disconnected mid-consult.**
- Rejoin from the same device if the session is still active.
- If rejoin fails, the provider should complete documentation for the
  partial session and schedule a follow-up if needed.
- Report recurring disconnections — they often indicate network issues at
  the consult endpoint.

**An inmate couldn't participate due to custody constraints.**
- Cancel or reschedule the session in the dashboard.
- Document the reason so the referral can be re-queued appropriately.

## Documentation

**A visit note is missing after a completed session.**
- Providers must complete documentation before session closure. Check if
  the session status is still "in progress."
- If the session shows complete but no note is visible, contact support
  with the session ID.

**We need to amend a visit note.**
- Contact [support@veridianhp.com](mailto:support@veridianhp.com) with the
  session ID and requested amendment. Amendments are logged in the audit
  trail per HIPAA requirements.

## Still stuck?

Email [support@veridianhp.com](mailto:support@veridianhp.com) with:

- Your facility name
- Session or referral ID (if applicable)
- A short description of the issue and what you've already tried

The Veridian team responds during business hours (US Central). For
security concerns, email [security@veridianhp.com](mailto:security@veridianhp.com)
directly.
