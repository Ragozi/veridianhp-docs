# Deployment and kiosk

Veridian Motion ships as a **Progressive Web App (PWA)** installable on
facility tablets. Backend services use Lovable Cloud / Supabase.

## PWA installation

Install via browser **Add to Home Screen / Install**. PWAs launch full-screen
for supervised patient intake stations.

<Note>
  **Limited offline.** Submit and photo upload require network connectivity.
</Note>

## App-level kiosk hardening (built today)

| Control | Behavior |
| --- | --- |
| **No external navigation** | External links blocked in kiosk context |
| **Strict CSP** | Limits script and resource loads |
| **Self-hosted fonts** | No third-party font CDN loads |
| **Idle auto-reset** | Patient session clears after inactivity |
| **In-app messaging only** | No SMS/email egress |

## Agency-provided lockdown

True locked-kiosk operation requires **agency MDM**:

| Control | Owner |
| --- | --- |
| MDM (single-app mode, disable browser) | Agency IT |
| Network firewall | Agency IT |
| Physical supervision | Custody staff |
| **CJIS / FedRAMP / GovCloud** hosting | Agency |

<Warning>
  Veridian Motion is **HIPAA-aligned, not HIPAA-certified**; **not
  CJIS-authorized or FedRAMP-authorized**. The **hosting agency** provides
  compliant environment, MDM, network controls, and personnel screening.
</Warning>

## Network

Kiosk devices need **HTTPS** outbound to the application host and Supabase
endpoints.

## Pilot vs production

Use separate environments for pilot. Synthetic or de-identified data until BAA
and agency hosting are in place.
