# Roles and access

Veridian Motion uses **role-based access control (RBAC)** backed by Supabase
Auth and **row-level security (RLS)** on every table. Staff roles are granted
**only by an admin**.

## Staff roles (built today)

| Role | ID | Access |
| --- | --- | --- |
| **Physical Therapist** | `pt` | Assigned assessments — review, approve, notes, messaging |
| **Administrator** | `admin` | Patient enrollment, assignment, role grant/revoke, MFA reset, roster |
| **IT Admin** | `it_admin` | Security settings, compliance views, technical administration |

See [Role permissions](/veridian-motion/role-permissions) for the full matrix.

<Note>
  **Rolling out:** multi-facility isolation, facility switching, and a
  **`warden`** facility-oversight role (operational metrics and safety
  escalations only — **no clinical PHI**). See
  [Roadmap](/veridian-motion/roadmap). Built today: `pt`, `admin`, `it_admin`
  only.
</Note>

<Note>
  Patient access is separate: patients authenticate with **Patient ID + PIN**,
  not staff roles. Patients see only their own submissions and approved/shared
  content.
</Note>

## Staff authentication

Staff accounts use:

1. **Email + password**
2. **TOTP MFA** — authenticator app; available on all staff accounts, **enforced
   per deployment**

There is no staff SSO (SAML/OIDC) in the current release. See
[Roadmap](/veridian-motion/roadmap).

## Role provisioning

Admins manage roles at **`/staff`**:

- **Grant** — assign `pt`, `admin`, or `it_admin` to a staff account
- **Revoke** — remove a role; access ends on next session
- View role assignments and MFA status

Role changes are recorded in the **hash-chained audit log**.

## Break-glass MFA reset

If a staff member loses their authenticator device, an **admin** or
**it_admin** may **reset MFA**. The user must re-enroll at next sign-in. MFA
resets are audit-logged.

## Patient credentials

| Action | Who | Result |
| --- | --- | --- |
| **Enroll patient** | `admin` | Patient ID + initial PIN returned to staff |
| **Reset patient PIN** | `admin` | New PIN shown once for handoff |
| **Sign in** | Patient at kiosk | Access to own submissions only |

Patients cannot change their PIN without staff assistance.

## What the app enforces vs. agency provides

| Control | Veridian Motion | Agency |
| --- | --- | --- |
| Role-based UI and API access | Yes | — |
| RLS on every table | Yes | — |
| TOTP MFA (per deployment) | Yes | — |
| Hash-chained audit log | Yes | — |
| Field-level PHI encryption | Yes | — |
| Brute-force login lockout | Yes | — |
| CJIS / FedRAMP hosting | — | Yes |
| MDM kiosk lockdown | — | Yes |
| Personnel screening | — | Yes |
| Physical kiosk supervision | — | Yes |
