# Status and workflow

Each **assessment** (one patient portal submission) moves through a defined
workflow. Staff and patients see **different labels** for the same underlying
state.

## Status map

| Staff status | Patient sees | Meaning |
| --- | --- | --- |
| **new** | **Received** | Submitted; server processing AI plan for PT review |
| **Awaiting assignment** | **Awaiting assignment** | Ready for admin to assign a PT |
| **Under review** | **Under review** | Assigned PT is reviewing |
| **reviewed** (approved) | **Reviewed** | PT approved; plan and shared notes visible to patient |

<Note>
  Patients never see staff-only labels like **new**. They never see AI output
  until status is **Reviewed** after PT approval.
</Note>

## Workflow diagram

```
[Admin enrolls patient] → Patient ID + PIN
        ↓
[Patient portal intake] → submit
        ↓
Staff: new / Received  →  AI generates PT-reviewable plan (dashboard only)
        ↓
Staff: Awaiting assignment  ←  Admin assigns PT
        ↓
Staff: Under review  ←  PT reviews and approves
        ↓
Patient: Reviewed + approved plan + shared notes
```

## Who acts when

| Step | Role | Action |
| --- | --- | --- |
| Enroll patient | **admin** | Manual enrollment; hand off Patient ID + PIN |
| Portal intake | **Patient** (custody escort) | Body map, complaint, questionnaire, photos; submit |
| Assign case | **admin** | Move from queue to assigned PT |
| Review / approve | **pt** (assigned) | Red flags first → edit plan → approve |
| Reset PIN | **admin** | Reissue credentials from patient record |
| Monitor kiosk | **Custody staff** | Physical supervision |

## Post-submit processing

After the patient submits:

1. The assessment record is created; the patient sees **Received** and
   confirmation (no AI content).
2. **Server-side AI** generates an immediate **PT-reviewable exercise plan**
   (name, why, steps, dosage, cautions) — visible only in the dashboard.
3. The submission enters the **assignment queue** when ready for PT assignment.
4. If processing is delayed beyond 30 minutes, contact
   [support@veridianhp.com](mailto:support@veridianhp.com).

## Multiple submissions

A patient may submit **more than one assessment** over time. Each appears in
**My plan / progress** with its own status.
