# Role permissions

Veridian Motion has **three staff roles** plus separate **patient** kiosk
access. This page reflects permissions **as built today** for a
**single-facility** deployment.

<Note>
  Staff require **email + password** and **TOTP MFA** (enforced per deployment).
  Roles are granted only by an **admin**.
</Note>

## Permission matrix

| Action | pt | admin | it_admin |
| --- | --- | --- | --- |
| Sign in to dashboard | Yes | Yes | Yes |
| View patient roster | Assigned + policy | Yes | No |
| Enroll patient (manual) | No | Yes | No |
| Reset patient PIN | No | Yes | No |
| View assignment queue | Assigned | Yes | No |
| Assign case to PT | No | Yes | No |
| Review / approve assessment | Yes (assigned) | No | No |
| View AI output / red flags | Yes (assigned) | Yes | No |
| Edit and release exercise plan | Yes (assigned) | No | No |
| PT notes | Yes (assigned) | No | No |
| Export assessment PDF | Yes | Yes | No |
| Clinician-to-patient messaging | Yes (assigned) | Policy | No |
| Grant / revoke staff roles | No | Yes | No |
| Break-glass MFA reset | No | Yes | Yes |
| Audit log | No | Yes | Yes |
| Security / compliance settings | No | Limited | Yes |

## Role summaries

### Physical Therapist (`pt`)

Primary **clinical user**. Reviews **assigned** assessments: red flags, intake,
AI exercise plan (name / why / steps / dosage / cautions). Edits, **approves**,
adds notes, messages patients, exports PDFs. Cannot enroll patients or grant
roles.

### Administrator (`admin`)

**Operational owner** for the facility. Manually enrolls patients, assigns
cases, resets PINs, grants/revokes roles, performs MFA reset, views roster and
audit logs. Does not replace PT clinical sign-off unless also holding `pt`.

### IT Admin (`it_admin`)

**Technical administration** — security settings, compliance tooling, MFA
break-glass with admin. No clinical review or patient enrollment by default.

## Patient (kiosk — not a staff role)

| Action | Allowed |
| --- | --- |
| Sign in with Patient ID + PIN | Yes |
| Complete patient portal intake | Yes |
| View own submission status | Yes |
| View approved plan after PT approval | Yes |
| View PT notes marked **shared with patient** | Yes |
| Reply in secure messaging threads | Yes |
| View AI output before PT approval | **No** |
| Self sign-up or change PIN | **No** |
| Access other patients' data | **No** |

## Enforcement

1. **Application UI and API** — role checks on staff actions
2. **Row-level security** — patient and staff queries scoped at the database;
   cross-patient access denied even if UI is bypassed
3. **Hash-chained audit log** — sensitive actions recorded without PHI in payload
