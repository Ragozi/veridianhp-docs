# Data model & privacy

Veridian Motion uses Supabase Postgres with **row-level security (RLS) on
every table**, **field-level PHI encryption**, and a **hash-chained audit
log**.

**HIPAA-aligned, not HIPAA-certified.** A **BAA** is required with the
health provider before production PHI.

## Core entities

| Entity | Description | PHI |
| --- | --- | --- |
| **Patient** | Manually enrolled — Patient ID, internal ID | Yes |
| **Assessment** | One portal submission (body map, complaint, questionnaire, photos, AI plan) | Yes |
| **Photos** | Up to four posture images per assessment | Yes |
| **Questionnaire responses** | Plain-language intake answers | Yes |
| **Exercise plan** | AI draft and PT-approved plan (name, why, steps, dosage, cautions) | Yes |
| **PT notes** | Clinical notes; optional share-with-patient flag | Yes |
| **Messages** | Append-only clinician ↔ patient thread | Yes |
| **Staff roles** | `pt`, `admin`, `it_admin` assignments | Metadata |
| **Audit log** | Hash-chained action record (no PHI in payload) | No PHI in logs |

## Data flow

```
Manual enroll (admin) → Patient ID + PIN
       ↓
Patient portal → Assessment (body map, complaint, questionnaire, photos)
       ↓
Server AI → PT-reviewable exercise plan (dashboard only)
       ↓
Assign (admin) → PT review → Approve
       ↓
Patient view → Approved plan + shared notes only
```

## Row-Level Security

| Actor | Access |
| --- | --- |
| **Patient** | Own record, own assessments/messages; plans **only after PT approval**; notes **only if shared** |
| **PT** | Assigned assessments and related patients |
| **Admin** | Roster, assignment, enrollment, roles for this facility |
| **IT admin** | Security/compliance views per role |

Cross-patient access is denied at the database layer.

<Note>
  **Multi-facility isolation is in progress.** Current deployments are
  single-facility. See [Roadmap](/veridian-motion/roadmap).
</Note>

## Photos

- Stored in a **private** bucket
- Authorized staff access only; patients do not receive storage URLs for other
  patients' images

## Session and client storage

- **No PII in browser `localStorage`** on kiosk
- Patient session state is **in-memory**; idle auto-reset clears it

## Audit log

Hash-chained, append-only entries for: enroll, assign, review/approve, notes,
roles, MFA reset, messages. **No PHI in audit payload.**

## Messaging

Append-only, monitored, internal only — part of the medical record under RLS.

## Agency vs. application

| Topic | Veridian Motion | Agency |
| --- | --- | --- |
| RLS, encryption, audit log | Yes | — |
| BAA | Design supports | Execute |
| CJIS / FedRAMP / GovCloud hosting | — | Yes |
| MDM and physical kiosk controls | App hardening | Device management |
