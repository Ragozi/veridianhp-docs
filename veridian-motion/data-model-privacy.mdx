# Data model & privacy

Veridian Motion uses **row-level security (RLS) on every table**, scoped by
**facility membership**, plus **field-level PHI encryption** and a
**hash-chained audit log**.

**HIPAA-aligned, not HIPAA-certified.** A **BAA** is required before production PHI.

## Core entities

| Entity | Description | PHI |
| --- | --- | --- |
| **Patient** | Manually enrolled — Patient ID, facility membership | Yes |
| **Assessment** | Portal submission (body map, complaint, questionnaire, photos, AI plan) | Yes |
| **Photos** | Up to four posture images | Yes |
| **Questionnaire responses** | Plain-language intake answers | Yes |
| **Exercise plan** | AI draft and PT-approved plan | Yes |
| **PT notes** | Clinical notes; optional share-with-patient flag | Yes |
| **Messages** | Append-only clinician ↔ patient thread (safety-screened) | Yes |
| **Communication Record** | Admin-only immutable messaging export | Yes (controlled access) |
| **Staff roles** | `pt`, `admin`, `warden`, `it_admin` + facility membership | Metadata |
| **Audit log** | Hash-chained actions (no PHI in payload) | No PHI in logs |

## Multi-facility isolation

- Each facility's **patients and staff** are isolated through **RLS** by
  **facility membership**
- **`pt`** — clinical data for assigned patients **within their facility**
- **`warden`** — **operational** data only (counts, activity, escalations) — **no clinical PHI**
- **`admin`** — operational and clinical admin per facility assignment
- Staff with multiple facilities **switch context** in the dashboard; queries
  remain facility-scoped

## Row-Level Security (summary)

| Actor | Access |
| --- | --- |
| **Patient** | Own record only; plans after PT approval; notes if shared |
| **PT** | Assigned patients **in their facility** |
| **Admin** | Roster, assignment, enrollment, Communication Record (facility-scoped) |
| **Warden** | Operational metrics and escalations — **no clinical PHI** |
| **IT admin** | Security/compliance views per role |

## Communication Record

- **Admin-only**, **immutable** export of messaging threads (PDF)
- **Every access** audit-logged in the hash-chained log
- Not available to **`warden`** or **`pt`** by default

## Audit log

Hash-chained entries for: enroll, assign, review/approve, notes, roles, MFA
reset, messages, **Communication Record access**. No clinical content in audit
payload.

## Agency vs. application

| Topic | Veridian Motion | Agency |
| --- | --- | --- |
| RLS, encryption, audit log | Yes | — |
