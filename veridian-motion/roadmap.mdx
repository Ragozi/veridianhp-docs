# Roadmap

This page lists capabilities that are **not yet available** in every
deployment, or are **planned** for a future release. For what is **built and
live today**, see [Platform overview](/veridian-motion/overview).

<Warning>
  Treat **Rolling out** items as available only when Veridian Health Partners
  confirms activation for your facility.
</Warning>

## Rolling out

| Capability | Description |
| --- | --- |
| **Admin analytics / ROI dashboard** | Per-facility, **admin-only** reporting on engagement and outcomes. Separates **measured** metrics from clearly labeled **estimated** cost figures; shows **no individual PHI**; any staff-wellness-related figures appear only in **aggregate / de-identified** form. See [Admin analytics](/veridian-motion/admin-analytics). |

## Planned

| Capability | Description |
| --- | --- |
| **Staff SSO (SAML / OIDC)** | Single sign-on via agency identity providers (e.g., Microsoft Entra ID, Google Workspace, Okta) with MFA and IdP group → Motion role mapping |
| **Bulk roster provisioning (CSV import)** | Admin upload of a roster CSV to create/update patients and issue PINs at scale |
| **JMS / booking-system integration** | Live or scheduled sync with jail management systems (e.g., Tyler, Spillman/Motorola Solutions, CentralSquare, Securus) — auto-provision on booking, deactivate on release |
| **Scheduled SFTP roster feed** | Daily roster file drop for automated reconcile |
| **EHR / EMR integration** | Automated clinical data exchange with external systems |
| **Partner API** | Programmatic access for third-party integrations |
| **Live video consults** | Synchronous telehealth sessions |

## Offerings (not a finished product claim)

| Offering | Description |
| --- | --- |
| **[Officer & Staff Wellness](/veridian-motion/officer-staff-wellness)** | Veridian Health Partners' **approach** to correctional **employee** wellness — voluntary, confidential movement and recovery content for staff. Not a certified or fully deployed product module; separate from patient care. |

## Explicitly out of scope (public docs)

- Medical billing / coding workflows
- AI output shown to patients before PT review
- Self-service patient or staff sign-up
- External messaging (SMS, email to patients)
- **AI behavioral-health, psychiatric, mood, or emotional-assessment modules** that interact with or assess a person psychologically (unreleased prototype — not documented)
